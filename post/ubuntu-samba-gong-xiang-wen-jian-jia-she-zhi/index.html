ubuntu samba 共享文件夹设置

1. 无密码共享文件夹搭建

  1. 使用apt-get安装samba

    sudo apt-get update # 更新apt-get软件列表

    sudo apt-get install samba # 安装samba

    [注] apt-get update 更新软件列表，apt-get upgrade为更新软件（慎用，更新后可能会出现软件和驱动不兼容的情况）

  2. 设置samba共享文件夹

    1. sudo mkdir /home/xiewr/sambashare # 创建共享文件夹
    2. sudo chmod 777 /home/xiewr/sambashare # 设置文件夹权限使访客用户有读写权限

  3. 编辑 /etc/samba/smb.conf配置文件，追加下面代码

    [sambashare]
   comment = share folder of AI Group	# 说明字段
   path = /home/xiewr/sambashare		# 分享文件夹路径 
   browseable = yes 					# 指定该共享是否可以浏览
   read only = no						# 设置可读写
   guest ok = yes						# 设置访客可访问

  4. 重启samba服务

    sudo service smbd restart

  5. 更新防火墙规则以允许Samba（设置一次即可）

    sudo ufw allow samba

2. 无密码共享文件夹搭建

  1. 添加samba账户（必须为linux系统中已有的账号，samba不使用linux原账号密码，因此这里需要设置一个新的密码）

    sudo smbpasswd -a username

    [注]在服务器的管理中，经常需要启动samba、mysql、elasticsearch、nginx服务，但又不希望以root的方式启动（有时候是强制性的，比如elasticsearch就禁止使用root账号启动服务），这时候就需要为它们创建单独的服务账号，同时为了安全性的考虑，可设置禁止登陆shell的账号作为samba账号。

  2. 设置valid users选项为1中的账号（这里为linsen-smb）

    [sambashare]
   comment = share folder of AI Group	# 说明字段
   path = /home/xiewr/sambashare		# 分享文件夹路径 
   browseable = yes 					# 指定该共享是否可以浏览
   read only = no						# 设置可读写
   valid users = linsen-smb				# 设置可访问samba用户

  3. 重启samba服务

    sudo service smbd restart

  4. 更新防火墙规则以允许Samba（设置一次即可）

    sudo ufw allow samba

3. 可能出现的问题

  1. 连接时出来这个提示: 不允许一个用户使用一个以上用户名与一个服务器或共享资源的多重连接。中断与此服务器或共享资源的连接，然后在试一次

    原因：

    ​ 现在使用的机器曾经与当前服务器有过连接，连接断开后操作系统未将连接状态清除。

    解决方案：

    1. 打开cmd命令窗口：
    2. 输入命令：net use查看你当前与网络资源的连接，例如驱动器映射、IPC连接。
    3. 用net use * /del /y命令中断开所有连接.
    4. 在重新登录就可以连接上了。
